# terraform-provider-shopify

## Build & Test

```bash
go build -o terraform-provider-shopify    # Build binary
go test ./...                              # Run tests
go generate ./...                          # Regenerate docs (tfplugindocs)
```

## Project Structure

```
main.go                              # Entry point, provider server
internal/provider/
  provider.go                        # Provider config (store_url, access_token, api_version)
  client.go                          # GraphQL HTTP client
  queries.go                         # GraphQL query/mutation strings
  product_resource.go                # shopify_product CRUD
examples/                            # Example .tf files (used by tfplugindocs)
docs/                                # Auto-generated by tfplugindocs (committed)
```

## Conventions

- **Framework**: terraform-plugin-framework (NOT the older SDKv2)
- **API**: Shopify GraphQL Admin API (NOT REST)
- **Module path**: `github.com/sophotechlabs/terraform-provider-shopify`
- **Registry address**: `registry.terraform.io/sophotechlabs/shopify`

## Shopify API Quirks

- Product creation auto-creates a default variant — variant fields update it after product create
- GraphQL IDs are GIDs: `gid://shopify/Product/123`, `gid://shopify/ProductVariant/456`
- Status values are uppercase: `ACTIVE`, `DRAFT`, `ARCHIVED`
- Variant prices are strings, not numbers: `"29.99"` not `29.99`
- `compareAtPrice`, `sku`, `barcode` are nullable — use `*string` in Go, `types.StringNull()` in framework
- Rate limit: 50 points/second for GraphQL (not yet handled in client)

## Adding a New Resource

1. Create `internal/provider/<resource_name>_resource.go`
2. Add GraphQL queries to `queries.go`
3. Register in `provider.go` → `Resources()` method
4. Add example in `examples/resources/shopify_<resource>/resource.tf`
5. Run `go generate ./...` to update docs
